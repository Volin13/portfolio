Web Store

Сайт інтернет магазину електронних та побутових товарів. Був натхненний відео з YouTube каналу Ulbi TV, мій перший fullstack проект. Крім основи, взятої з каналу, було реалізовано: корзину, секцію коментарів, сторінку оформлення товару, фільтрацію товарів, відслідковування статусу замовлення, редагування сторінки товару, прелоадери, загалом покращений дизайн інтервейсу та рішення для кращого UI. Реалізовано збереження зображень в хмарному середовищі Cloudinary, у формі замовлення товару використано NovaPostAPI для списку областей/міст/відділень. Для безпеки використаний механіз обмiну access та reset токенів. Проект велийкий, під час його розробки я дізнався багато нового та покращив свої навички і розуміння web-розробки. Пре переході за посиланням доведеться почекати мінімум 50 секунд - це особливість використання безкоштовного деплою на сервісі Render.

GitHub Web

Головна сторінка

Містить у собі список товарів з відмітками про наявність та знижки, їх назви та ціну. Реалізована фільтрація товарів за брендами, типами, ціною та рейтингом (в зростаючиму/спадаючому порядку). У верхній частині екрану розміщене поле пошуку за назвою товару. У нижній частині розташований блок посилань на соціальні мережі.

Прелоадери-заглушки

Під час завантаження даних з серверу відображаються прелоадери-заглушки для тексту та зображень

Фільтрація за брендами + ціною/рейтингом

Робота фільтрації товарів за брендами. Натиснутий бренд стає першим в списку брендів та міняє колір. Список брендів можна розгорнути натисненням на кнопку. В боковому меню розташовані свіч-перемикачі для фільрації за ціною/рейтингом.

Фільтрація за типами + назвою

Робота фільтрації товарів за типами. Натиснутий тип стає першим в списку типів та міняє колір. Список типів можна розгорнути натисненням на кнопку. При наведені на іконку пошуку розгортається полевведення назв.

Бокове меню

При натисканні кнопки меню в хедері сторінки розгортається бокове меню. Меню містить розділи: інфо користувача (та його зміна), сторінку для відстеження статусу покупки, меню адміністратора (тільки для аккаунту адміністратора), кнопку виходу з аккаунту

Меню користувача

Меню користувача відображає нікнейм та аватар користувача, юзер може змінити їх. При зміні аватара відбувається валідація на тип кантинки та її розмір. При зміні нікнейму відбувається валідація (користувач не може вводити спеціальні символи), також проходить перевірка на унікальність нікнейму.

Кошик

Кнопка кошика показує кількість товарів, які знаходяться в кошику, сама кількість зберігається в session storage. При натисканні на кнопку відкривається модальне вікно з списком покупок, кількістю та сумою до сплати. Є можливість очистити кошик, змінити кількість окремого товару або видалити товар зі списку. При натисканні на кнопку оформлення замовлення, користувач перейде на сторінку з формою, яку потрібно заповнити для завершення купівлі.

Сторінка девайсу

Сторінка девайсу містить в собі зображення девайсу, назву, ціну, кнопку для замовлення, характеристики девайсу та секцію коментарів. При натисканні на фото товару відкриється модальне вікно з слайдером фотографій девайсу більшого формату. Характеристики девайсу можна розгорнути, або проскролити. Кнопка редагування доступна тільки адміністратору сайта. При натисканні на кнопку замовлення товару змінюється картинка (яка залежить від кількості товарів у кошику), та з'являється нотифікація у правому верхньому куті екрану.

Вікно редагування девайсу

Вікно редагування девайсу дає можливість адміністратору редагувати: назву товару, приналежність до бренду/типу, наявність плашки що індикує знижку/наявність, ціну/нову ціну зі знижкою, рейтинг, редагувати список характеристик, головне зображення та другорядні зображення.

Адмін панель

Адмін панель містить в собі кнопки що відкривають модальні вікна з можливістю: додавання нового типу, додавання бренду, пристрою, переглядом коментарів їх редагуванням та видаленням,перегляд зроблених замовлень

Історія замовлень користувача

Історія замовлень користувача показує замовлення користувача, їх статус (ідинтифікація за кольором), також при натисканні на замовлення можна переглянути замовлення та данні введені при оформленні

Коментарі користувачів

Це модальне вікно містить всі коментарі на сайті. Адміністратор може переглядати коментарі, редагувати їх або видаляти. Також можливий пошук коментарів по даті.

Форма оформлення замовлення

Форма оформлення замовлення містить у собі поля: ім'я, прізвище, імейл, телефон, область, місто, відділення, додаткове поле з коментарем. Всі поля мають відповідну валідацію. Для випадаючих списків у розділі адреси доставки було використано NovaPostAPI. Список областей зтягується раз у пів року (за наявності старого) та зберігається у локальному середовищі.

Валідація форми оформлення замовлення

Кожне поле має відповідні правила валідації, окрім поля коментаря, де присутнє тільки обмеження по довжині. Поля адреси доставки мають бути обрані з випадаючого списку, обов'язково.

Слайдер зображень девайсу

При натисканні на зображення товару на сторінці товару, вічиняється модальне вікно з слайдером зображень більшого формату.

  • react

    Бібліотека JavaScript для створення інтерфейсів користувача, яка дозволяє будувати компоненти та ефективно управляти станом для динамічних веб-додатків.

  • formik

    Бібліотека для управління станом та валідацією форм у React, яка спрощує роботу з формами та їх обробку.

  • yup

    Бібліотека для валідації даних у JavaScript, яка дозволяє створювати схеми для перевірки форм та об'єктів на відповідність визначеним правилам.

  • axios

    Бібліотека для управління станом у JavaScript/React-додатках, яка забезпечує автоматичне відстеження змін у даних і оновлення інтерфейсу користувача.

  • mobx

    бібліотека для управління станом у JavaScript/React-додатках, яка забезпечує автоматичне відстеження змін у даних і оновлення інтерфейсу користувача.

  • bootstrap

    Бібліотека компонентів для React, яка забезпечує інтеграцію з Bootstrap для створення адаптивного інтерфейсу користувача.

  • jwt-decode

    Бібліотека для декодування JSON Web Tokens (JWT) без необхідності валідації або запиту до сервера

  • dotEnv

    Бібліотека для завантаження змінних середовища з файлу .env у Node.js додатки, що дозволяє зберігати конфіденційні дані окремо від коду.

  • express

    Фреймворк для Node.js, який спрощує створення серверних додатків та обробку HTTP-запитів.

  • googleapis

    Офіційний клієнт для доступу до Google APIs, який дозволяє взаємодіяти з різними сервісами Google, такими як Google Drive, Gmail, Google Sheets тощо, з Node.js.

  • jsonwebtoken

    Бібліотека для створення та перевірки JSON Web Tokens (JWT) у Node.js, яка забезпечує механізм аутентифікації та авторизації.

  • sequelize

    ORM (Object-Relational Mapping) для Node.js, який дозволяє взаємодіяти з реляційними базами даних (MySQL, PostgreSQL, SQLite, MSSQL) за допомогою JavaScript-об'єктів.

  • moment

    Бібліотека JavaScript для роботи з датами та часом, яка спрощує форматування, парсинг і маніпулювання датами

  • prop-types

    Бібліотека для перевірки типів властивостей (props) у компонентах React, що допомагає забезпечити коректність даних, що передаються в компоненти.

  • react-router-dom

    Бібліотека для реалізації маршрутизації в React-додатках, яка дозволяє створювати маршрути і управляти навігацією між різними компонентами та сторінками.
